RAG Is Not a Truth Machine

Retrieval narrows the evidence available to a language model. It does not guarantee that the right evidence was found or interpreted correctly.

By Alissa KingJune 20, 2026 · 7 min read

Retrieval-augmented generation improves the conditions under which a model answers. It does not remove the need to evaluate the answer.

I built Chat with Your Documents to make the pattern concrete. A user uploads a PDF, the app extracts and divides its text, retrieves passages related to a question, and gives those passages to Gemini as context for a response.

This is a meaningful improvement over asking a general model to answer from memory. The response is constrained by a user-selected source. But the system still contains several stages where relevant evidence can be lost, distorted, or overinterpreted.

Retrieval is an information bottleneck

The app divides extracted text into 500-word chunks with 50 words of overlap. It embeds each chunk locally with the normalized all-MiniLM-L6-v2 sentence-transformer model. A question is embedded the same way, and the five chunks with the highest cosine similarity are selected.

PDF → chunks → embeddings → top five excerpts → answer

By the time the language model sees the document, most of the document is absent. That is the point—focused context is more useful than an indiscriminate wall of text—but it means retrieval quality sets an upper bound on answer quality.

If the relevant passage ranks sixth, the generator never receives it. If a table is poorly extracted, no embedding can restore its structure. If the question uses terminology far removed from the document, semantic similarity may select adjacent rather than decisive evidence.

Chunking creates boundaries the document did not have

Fixed-size chunks are simple and effective, but meaning does not arrive in 500-word units. A definition may appear in one section and its exception in another. A heading may apply to several paragraphs. Footnotes, tables, and page layout may carry relationships that plain extracted text does not preserve.

Overlap reduces the chance that a sentence is cut away from its immediate context. It does not solve relationships that span pages or depend on document structure. More advanced systems may use headings, pages, tables, metadata, reranking, or query expansion. Each addition creates new choices to test.

“Answer only from context” is a control, not a guarantee

The app’s system instruction tells Gemini to use only the supplied excerpts and to say when the answer is not present. That is an important behavioral constraint. It is not a mathematical proof that every claim is supported.

The generator may combine excerpts incorrectly, infer beyond them, or state an ambiguous passage too confidently. The retrieved text itself can contain instructions that were never intended for the model. In higher-stakes systems, source display, quotation-level citations, adversarial testing, and explicit refusal evaluation should accompany the prompt.

Two failures to separate

Retrieval failure: the system did not supply the needed evidence. Generation failure: the evidence was supplied, but the answer misused it. Improvement depends on knowing which one occurred.

Conversation memory can move the question

The app retains the last 20 conversation turns so follow-up questions can remain coherent. That helps users ask “What does that mean?” without restating the subject. It also introduces another source of influence: previous answers may shape later questions even if an earlier response was incomplete.

Evaluation should therefore include multi-turn conversations, document changes, ambiguous references, unsupported questions, and cases where the correct response is “the document does not say.” Abstention is a feature when the evidence is missing.

Grounded is not the same as correct

A RAG system can make answers more inspectable by tying them to a bounded source. Its strongest promise is not truth; it is provenance. Users can compare the answer with the passages that informed it—provided the interface exposes those passages clearly.

The responsible claim

RAG can reduce unsupported generation and improve access to source material. It cannot certify the source, guarantee retrieval completeness, or replace review where errors carry meaningful consequences.

The next useful improvements to this project are therefore evaluative: display source excerpts with page metadata, build a question-and-answer test set, measure retrieval recall separately from answer quality, and test refusal behavior. Better generation may help. Better evidence about the full pipeline will help more.
